# 社交急救包 - V1.2 开发规格书

> 本文档供 Claude Code 开发使用，包含 V1.2 全部功能的技术实现规格。
> PRD 文档：https://my.feishu.cn/docx/EWS7dBDFUoRK82x0PPTcwc0XnYf

---

## 一、项目概览

- **项目代号**：社交急救包 (socialSOS)
- **产品形态**：微信小程序（原生开发）
- **后端**：微信云开发（云函数 + 云数据库 + 云存储）
- **V1.0 策略**：全部免费，无付费墙（支付代码预留但不开启）
- **设计风格**：趣味游戏化，像"闯关游戏"而不是"刷题工具"

---

## 二、技术栈

| 环节 | 方案 | 说明 |
|------|------|------|
| 小程序框架 | 微信小程序原生开发 | 性能最好，审核最稳 |
| 后端服务 | 微信云开发 | 免服务器、免域名备案 |
| 数据库 | 微信云数据库 | questions + users 两个集合 |
| 云函数 | Node.js | 抽题逻辑、用户数据、排行榜 |
| 支付 | 微信小程序内支付 | V1 预留接口，不开启 |
| AI 编码 | Claude Code | 生成 90% 代码 |

## 三、页面结构（2个 Tab）

```
小程序
├── TabBar
│   ├── 首页 (pages/index/index)        ← 核心答题页
│   └── 我的 (pages/profile/profile)    ← 个人中心
├── pages/question/question            ← 答题交互页
├── pages/character/character           ← 角色选择页（首次进入）
├── pages/series/series                 ← 微剧系列页
├── pages/quiz/quiz                     ← 社恐测评页
├── pages/favorites/favorites           ← 收藏列表页
├── pages/cheatsheet/cheatsheet         ← 小抄集页
├── pages/share/share                   ← 分享海报页（canvas生成）
└── pages/setting/setting               ← 设置页
```

### 用户流程

```
首次打开 → 角色选择页（选i人角色）→ 首页
                                          ↓
                        首页：今日推荐卡片 / 场景分类标签
                                          ↓
                        点击答题 → 答题页（多题型交互）
                                          ↓
                        选完 → 反馈动效（社死/社牛）
                                          ↓
                        展示最佳答案 + 课后小抄
                                          ↓
                        点"下一题" → 继续刷 / 回首页
```

## 四、功能模块详细规格

### 4.1 角色选择（首次进入）

4个 i 人角色，RPG 选职业风格：

```javascript
const CHARACTERS = [
  { id: 'work-ghost', name: '职场小透明', emoji: '🫥', desc: '开会永远坐在角落' },
  { id: 'party-ninja', name: '聚会隐形人', emoji: '🥷', desc: '聚会存活率：靠墙' },
  { id: 'love-mute', name: '恋爱哑巴', emoji: '💬', desc: '暗恋三年没开口' },
  { id: 'campus-wallflower', name: '校园壁花', emoji: '🏫', desc: '自我介绍能卡3分钟' }
];
```

- 首次进入强制选择（不可跳过）
- 选完后存入用户数据，可后续在"我的"页面修改
- 角色头像显示在个人中心和答题页顶部

### 4.2 首页

**顶部区域：**
- 用户角色头像 + 昵称（默认微信昵称）
- "社交能量值"进度条（XP条）
- 段位图标 + 段位名称

**场景分类标签（横向滚动）：**
```javascript
const CATEGORIES = [
  { id: 'work', name: '🏢 职场生存', count: 50 },
  { id: 'party', name: '🎉 社交聚会', count: 40 },
  { id: 'love', name: '💕 恋爱沟通', count: 30 },
  { id: 'campus', name: '🎓 校园生活', count: 30 },
  { id: 'daily', name: '🛒 日常人际', count: 30 },
  { id: 'family', name: '🧧 亲戚应酬', count: 20 },
  { id: 'all', name: '🎲 随机挑战', count: 0 }  // count不显示
];
```
- 点击标签筛选对应分类题目
- 选中标签高亮，筛选结果实时更新
- "随机挑战"为默认选项

**今日推荐卡片：**
- 每天从题库中推荐一道"今日社死挑战"（地狱级 difficulty=3）
- 卡片展示：场景描述前半段 + "点击挑战"按钮
- 卡片带日期标记，每天自动更换

**微剧入口：**
- "追剧专区"横向卡片列表，展示 6 部微剧
- 每部微剧显示：名称 + 已解锁集数/总集数 + 进度条
- 点击进入微剧详情页

**底部练习数据：**
- 今日已练 X 题
- 连续 X 天（连续打卡天数）
- 总计已练 X 题

### 4.3 答题页（核心页面）

支持 6 种题型，根据 `questionType` 字段渲染不同交互：

#### 题型 1: 经典三选一 (choice) — 占比 40%

```
┌─────────────────────────────┐
│ 🏢 职场生存    ⭐⭐ 难度2     │
│─────────────────────────────│
│                             │
│ 周一下午三点的部门例会，你    │
│ 正低头假装记笔记。突然领导    │
│ 说："小王，这个方案你怎么看"│
│ 全组13双眼睛齐刷刷转向你... │
│                             │
│ （不是吧我什么都不知道啊）   │  ← 内心OS，灰色斜体
│                             │
│─────────────────────────────│
│ A. 嗯...我觉得都可以         │
│ B. 我没啥想法，听大家的吧    │
│ C. 不好意思打断一下...       │
│                             │
│         ❤️ 5s               │  ← 限时倒计时（可关闭）
└─────────────────────────────┘
```

**选择后反馈：**
- 选错：屏幕震动 + 社死动效 + "社死程度 ⭐⭐⭐⭐" 评星 + 毒舌吐槽
- 选对：金色光效 + "社牛！你今天又进步了！" 飘字 + 能量值 +1

**答案展示：**
- 显示所有风格答案（稳妥型🛡️ / 主动型🚀 / 高能型⚡）
- 用户可点击"我是这种风格"收藏自己的偏好
- 底部展示课后小抄（可展开/收起）
- "一键复制话术"按钮
- "收藏此题"按钮
- "下一题"大按钮

#### 题型 2: 填空补全 (fillBlank) — 占比 15%

```
场景描述（含内心OS）

下半句：___________

[输入框，用户输入答案]

提交后对比最佳答案，显示匹配度评分
```

#### 题型 3: 接力对话 (relay) — 占比 15%

```
第一轮：
  对方："小王，周末有空吗？团建。"
  你选：A/B/C

→ 根据选择，对方给出不同回应

第二轮：
  对方："行，那就这么说定了！"（或拒绝回应）
  你选：A/B/C

→ 最终评价
```

**数据结构扩展：**
```javascript
{
  "questionType": "relay",
  "rounds": [
    {
      "dialog": "对方说的话...",
      "options": [
        { "text": "选项A", "nextRoundIndex": 0, "score": 30 },
        { "text": "选项B", "nextRoundIndex": 1, "score": 80 },
        { "text": "选项C", "nextRoundIndex": 2, "score": 60 }
      ]
    },
    // round 1...
    // round 2...
  ]
}
```

#### 题型 4: 二选一抉择 (dilemma) — 占比 10%

```
两个选项都尴尬，选"更不尴尬"的那个

A. 沉默点头，假装听到了     [社死度：⭐⭐⭐]
B. 说"你说什么？我走神了"    [社死度：⭐⭐⭐⭐]

选完后揭晓：B 虽然社死，但比A好，因为...
```

#### 题型 5: 找茬题 (spotMistake) — 占比 10%

```
展示一段多轮对话（3-4句）

用户点击/高亮认为"最不该说的那句话"

正确标出问题句 → 解释为什么这句话有问题
```

**数据结构：**
```javascript
{
  "questionType": "spotMistake",
  "dialog": [
    { "role": "colleague", "text": "王哥，你最近是不是忙项目啊？" },
    { "role": "me", "text": "还好吧" },
    { "role": "colleague", "text": "看你天天加班，注意身体啊" },
    { "role": "me", "text": "也没天天加，你别瞎操心" },  // ← 问题句
    { "role": "colleague", "text": "..." }
  ],
  "mistakeIndex": 3,
  "explanation": "对方在关心你，'别瞎操心'直接把好意怼回去了..."
}
```

#### 题型 6: 话术拼图 (puzzle) — 占比 10%

```
打乱的话术片段，拖拽排序成正确顺序：

[不好意思] [我有个想法] [打断一下] [关于刚才的方案]

正确顺序：不好意思 → 打断一下 → 关于刚才的方案 → 我有个想法
```

**数据结构：**
```javascript
{
  "questionType": "puzzle",
  "correctOrder": [
    "不好意思",
    "打断一下",
    "关于刚才的方案",
    "我有个想法"
  ],
  "fullSentence": "不好意思打断一下，关于刚才的方案，我有个想法。"
}
```

### 4.4 限时挑战模式

- 答题页顶部 5 秒倒计时（心跳动画 ❤️❤️❤️）
- 超时自动判定"社死"，跳过该题
- 可在设置页关闭：`settings.enableTimer = false`
- 超时不算答题次数，不计入正确率

### 4.5 社死↔社牛进度条

- 首页和个人中心展示全局进度条
- 左端"💀 社死"，右端"🦸 社牛"
- 根据累计答题正确率计算位置：
  - `socialScore = correctRate * 100` （0-100）
- 进度条带动画过渡，每次答题后滑动更新

### 4.6 段位系统

```javascript
const RANKS = [
  { id: 'bronze', name: '青铜i人', minScore: 0, icon: '🥉', color: '#CD7F32' },
  { id: 'silver', name: '白银', minScore: 20, icon: '🥈', color: '#C0C0C0' },
  { id: 'gold', name: '黄金', minScore: 40, icon: '🥇', color: '#FFD700' },
  { id: 'diamond', name: '钻石社牛', minScore: 65, icon: '💎', color: '#B9F2FF' },
  { id: 'king', name: '王者', minScore: 85, icon: '👑', color: '#FF6B6B' }
];
```

- `minScore` 基于累计答题积分（答对+1，连续答对+1bonus）
- 段位升级时弹出升级动效 + 专属称号
- 段位卡可生成分享海报（canvas 绘制）

### 4.7 微剧系列

6 部微剧，每部 5 集，共 30 题：

```javascript
const SERIES = [
  {
    id: 'work-life',
    name: '《职场生存记》',
    desc: '从小透明到能在年会讲脱口秀',
    episodes: ['wp-e01', 'wp-e02', 'wp-e03', 'wp-e04', 'wp-e05']
  },
  {
    id: 'new-year',
    name: '《过年回乡历险记》',
    desc: '活着从亲戚包围圈逃出来',
    episodes: ['ny-e01', 'ny-e02', 'ny-e03', 'ny-e04', 'ny-e05']
  },
  // ... 其余 4 部类似
];
```

- 答完第 N 集解锁第 N+1 集（series_progress 字段记录）
- 每集结尾展示"下集预告"（下集场景描述前半段）
- 全部通关获得系列专属称号 + 通关海报
- 微剧页展示已解锁/未解锁状态

### 4.8 收藏功能

- 答题页"收藏"按钮，收藏的题目存入 users.favorites 数组
- "我的"页面入口 → 收藏列表页
- 收藏列表按分类分组展示
- 长按可取消收藏

### 4.9 小抄集

- 每道题的 `cheatsheet` 自动归集到小抄集
- 小抄集页按分类分组展示
- 每条小抄显示：`oneLineRule` + `rememberThis`
- 顶部显示收集总数："已收集 X 条小抄 🔥"
- 支持按分类筛选

### 4.10 社恐测评

10 道场景题，每道选 A/B/C：
```javascript
const QUIZ_QUESTIONS = [
  {
    "scene": "聚会中有人走过来跟你搭话，你第一反应是？",
    "options": [
      "微笑回应，试着聊几句",
      "假装看手机，希望对方知难而退",
      "僵住，脑子一片空白"
    ]
  },
  // ... 共10题
];
```

- 答完后根据选择计算结果（每个选项加权分）
- 生成社恐类型标签（如"回避型社恐""爆发型社恐""隐形社牛"等）
- 结果页带描述 + 建议 + 分享海报（canvas 生成，含小程序码）

### 4.11 分享海报

使用 canvas 绘制，支持两种场景：
1. **段位卡**：段位图标 + 称号 + 数据
2. **测评结果卡**：社恐类型 + 描述 + 小程序码

```javascript
// 海报规格
const POSTER = {
  width: 750,
  height: 1334,
  bgColor: '#1A1A2E',  // 深色背景，高级感
  textColor: '#FFFFFF',
  accentColor: '#E94560'  // 强调色
};
```

### 4.12 设置页

- 限时模式开关（默认开启）
- 音效开关（默认开启）
- 动效开关（默认开启）
- 清除本地缓存
- 关于我们
- 意见反馈（调用微信客服消息接口）

  // === 选项与答案 ===

  options: ["选项A", "选项B", "选项C"],

    {
      answer: "稳妥型回答...",

    },
    {

## 九、题库内容规范

### 9.1 场景描写规则

- **必须包含**：时间 + 地点 + 人数 + 感官细节
- **第一句**就要有画面感
- **内心OS**用灰色小字 + 括号展示
- 文案风格：轻松幽默，不沉重不焦虑

### 9.2 答案规范

- 每道题至少 2 种风格答案（稳妥型 + 主动型）
- 话术要"接地气、不油腻、能直接用"
- 避免：教科书式说教、过度PUA式高情商

### 9.3 反馈文案规范

- 毒舌但不伤人，幽默吐槽
- 星级评分（⭐1-5）直观展示社死程度
- 示例："这个回答，在场所有人同时低头看手机了..."

## 十一、注意事项

1. **敏感词规避**：小程序审核严格，避免"社恐""焦虑""抑郁"等医疗相关词，文案中使用"i人""内向""社交小白的自救指南"等替代
2. **支付功能**：代码中预留支付接口和付费状态字段，但 V1 默认所有用户 `unlocked: true`
3. **云开发免费额度**：月活1万以内完全免费，关注用量即可
4. **性能**：题库数据用云数据库，首次加载后本地缓存（wx.setStorage），减少云函数调用
5. **分享合规**：分享海报不含诱导分享文案，符合微信审核要求
